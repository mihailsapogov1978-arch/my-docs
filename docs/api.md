## Шаг 1. Заголовок и базовая информация

> Правило: Метод + путь + краткое название (2-4 слова)

`POST /api/v1/signup` — Регистрация пользователя

## Шаг 2. Таблица с фактами из main.go
> Правило: Пишем только то, что видим в коде. Если чего-то не знаем — помечаем

| Поле | Значение |
|------|----------|
| **Метод** | `POST` |
| **Путь** | `/api/v1/signup` |
| **Обработчик** | `handlers.UserSignup` |
| **Авторизация** | ❌ Не требуется (публичный маршрут) |
| **Параметры в пути** | Нет |

## Шаг 3: Описание (что делает эндпоинт)

> Правило: 1-2 предложения простым языком. Что происходит? Зачем это нужно?

**Описание**: 
Создаёт нового пользователя в системе. После успешной регистрации пользователь может войти через `/api/v1/signin`.

## Шаг 4: Параметры запроса (тело, JSON)

> Правило: Таблица — лучший формат. Если не уверены — добавляем пометку ⚠️

| Поле | Тип | Обязательный | Описание |
|------|-----|--------------|----------|
| `email` | string | Да| Email пользователя для входа |
| `password` | string | Да| Пароль (мин. 8 символов) |
| `username` | string | Да| Имя пользователя для отображения |

> ⚠️ *Примечание: точные поля требуют проверки в `handlers.UserSignup`*

## Шаг 5. Пример запроса

```json
{
  "email": "anna@example.com",
  "password": "MyPassword123!",
  "username": "anna_trader"
}
```
> **Правило**: Пример должен быть реалистичным (не `test/test`, а похожим на реальные данные)

## шаг 6. Пример успешного ответа (201 Created)

```json
{
  "message": "User registered successfully",
  "user_id": "usr_abc123"
}
```

> **Правило**: Указываем код статуса (200, 201, 400 и т.д.)

## Шаг 7: Возможные ошибки

| Код | Значение | Причина |
|-----|----------|---------|
| `400` | Bad Request | Не указаны обязательные поля |
| `409` | Conflict | Email уже зарегистрирован |
| `500` | Internal Server Error | Ошибка сервера |
