### 2. Техническая документация в Obsidian

**Короткий ответ:** Можно, но с оговорками.

#### ✅ Плюсы Obsidian для техдокументации:

| Преимущество | Почему важно |
|--------------|--------------|
| **Backlinks** | Автоматически показывает, где упоминается термин/концепция |
| **Graph view** | Визуализация связей между страницами — отлично для архитектуры знаний |
| **Быстрая навигация** | `[[Ссылки]]`, поиск, теги работают мгновенно |
| **Локальное хранение** | Markdown-файлы — полный контроль, резервное копирование просто |
| **Плагины** | Мощная экосистема для расширения функционала |

#### ❌ Минусы для больших мануалов:

| Проблема | Последствия |
|----------|-------------|
| **Нет встроенной публикации** | Нужно экспортировать или использовать сторонние инструменты |
| **Нет веб-интерфейса «из коробки»** | Пользователи не могут просто открыть ссылку и читать |
| **Ограниченная навигация** | Нет встроенного меню/сайдбара как в документации на сайте |
| **Поиск только для владельца** | Посетители не могут искать по документации |
| **Нет версионности** | Сложно поддерживать несколько версий продукта |
| **Масштабирование** | При сотнях страниц граф становится «спагетти», сложно ориентироваться |
