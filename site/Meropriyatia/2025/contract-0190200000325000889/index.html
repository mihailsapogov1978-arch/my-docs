    Реквизиты документа

    Приложение № 1 к государственному контракту № 889 от 04 марта 2025 г.

    Информация о лицензии

    Заказчик располагает неисключительным правом на использование программного обеспечения — Автоматизированная система бюджетного учета «Смета» (АС «Смета») (правообладатель ООО «НПО «Криста», ИНН 7707758779) без ограничения по сроку действия лицензии и количества пользователей/подключений.

    I. Общие сведения¶

    1.1. Наименование услуг¶

    Оказание услуг по миграции баз данных в единую программную платформу ГИС «Смета ЯНАО» (далее — услуги).

    II. Цели оказания услуг¶

    2.1. Цели оказания услуг¶

    Оказание услуг направлено на достижение следующих целей:

    • подключение Пользователей к функциям Системы;
    • обеспечение унификации ведения бюджетного (бухгалтерского) учета и отчетности в Субъектах бухгалтерского учета;
    • обеспечение единства и однократности ввода нормативно-справочной информации в Системе, основанных на использовании web-технологий;
    • снижение трудозатрат и повышение производительности ведения учета финансово-хозяйственной деятельности участников процессов ведения бюджетного (бухгалтерского) учета.

    III. Требования к оказанию услуг¶

    3.1. Состав услуг¶

    В состав оказываемых услуг по ТЗ входят:

    • оказание услуг по переносу в Систему и выверке базы данных из ППО Субъекта бухгалтерского учета;
    • подключение пользователей Субъекта бухгалтерского учета к Системе;
    • оказание услуг по подготовке пользователей Субъекта бухгалтерского учета к работе с Системой.

    3.2. Требования к соблюдению конфиденциальности информации¶

    Исполнитель обязан соблюдать конфиденциальность и обеспечивать безопасность информации, к которой он получает доступ в ходе исполнения ТЗ, и которая отнесена законодательством к категории конфиденциальной. Исполнитель обязан принимать предусмотренные законодательством меры по защите такой информации.

    Исполнитель не вправе каким-либо образом использовать вышеуказанную информацию, за исключением случаев, если это не противоречит законодательству, необходимо для оказания услуг, предусмотренных настоящим ТЗ, и отсутствуют иные способы оказания услуг, по согласованию с Субъектом бухгалтерского учета.

    3.3. Требования к переносу и выверке базы данных¶

    В рамках оказания услуг по переносу и выверке базы данных в Систему Исполнитель обязан произвести:

    Перенос данных¶

    • перенос в Систему данных из ППО с использованием стандартных (существующих на текущий момент в Системе) механизмов, по аналитическим признакам, соответствующим требованиям к аналитическому учету по счетам бухгалтерского учета, установленным приказом Минфина РФ от 01.12.2010 № 157н «Об утверждении единого плана счетов бухгалтерского учета для органов государственной власти (государственных органов), органов местного самоуправления, органов управления государственными внебюджетными фондами, государственных академий наук, государственных (муниципальных) учреждений и инструкции по его применению» (далее – Приказ 157н), учетной политики Субъекта бухгалтерского учета, с учетом данных, представленных в приложении № 1 к настоящему ТЗ;

    • перенос данных справочников и классификаторов, разработанных в соответствии с нормативными правовыми актами федеральных органов исполнительной власти, департамента финансов Ямало-Ненецкого автономного округа, требованиями учетной политики Субъекта бухгалтерского учета и необходимых для ведения (бюджетного) бухгалтерского учета Субъекта бухгалтерского учета;

    Регистрация и настройка пользователей¶

    • регистрацию пользователей Субъекта бухгалтерского учета в Системе;
    • создание учетных записей пользователям Субъекта бухгалтерского учета, разграничение прав доступа в Системе;

    Проверка корректности загруженных данных¶

    Проверка корректности загруженных данных на основе предоставленных Субъектом бухгалтерского учета регистров бухгалтерского учета:

    Регистр Форма Период
    Главная книга ОКУД 0504072 с начала отчетного года по дату передачи базы Исполнителю
    Оборотная ведомость по нефинансовым активам ОКУД 0504035 с начала отчетного года по дату передачи базы Исполнителю
    Инвентарные карточки учета нефинансовых активов ОКУД 0509215 сведения об инвентарных объектах, числящихся в учете на начало отчетного года и поступивших (выбывших) на дату передачи базы Исполнителю, с полной историей зарегистрированных по ним операций
    Инвентарные карточки группового учета нефинансовых активов ОКУД 0509216 сведения об инвентарных объектах, числящихся в учете на начало отчетного года и поступивших (выбывших) на дату передачи базы Исполнителю, с полной историей зарегистрированных по ним операций
    Оборотная ведомость ОКУД 0504036 в разрезе аналитических признаков, приведенных в Приложении №1 к настоящему ТЗ, сформированная за период с начала отчетного года по дату передачи базы Исполнителю
    Сводные ведомости по начислению заработной платы, удержаниям — за 2022 - 2023 год в помесячной разбивке, с группировкой (разбивкой) по источникам финансирования
    Книга учета бланков строгой отчетности — за период с начала отчетного года по дату передачи базы Исполнителю с диапазоном номеров (при наличии учета бланков строгой отчетности)
    Иные регистры бухгалтерского учета — необходимые для проверки загруженных данных

    Порядок предоставления регистров

    Указанные регистры предоставляются Субъектом бухгалтерского учета на дату предоставления базы данных исходного ППО в электронном виде на адрес Исполнителя.

    Для проведения проверки корректности загруженных данных могут привлекаться пользователи Системы (по согласованию).

    Организационное содействие¶

    Субъект бухгалтерского учета оказывает организационное содействие Исполнителю по вопросам предоставления базы данных ППО, в том числе предоставляет перечень дополнительных аналитических признаков и/или счетов субъекта бухгалтерского учета в адрес Заказчика непосредственно перед проведением работ по миграции баз данных в единую программную платформу.

    Порядок предоставления базы данных ППО¶

    Субъект бухгалтерского учета предоставляет дистанционный доступ Исполнителю к базе данных ППО, используя защищенный канал связи посредством программного обеспечения ViPNet Client, либо передает Исполнителю базу данных ППО, используя съемный носитель служебной информации ограниченного доступа в соответствии с положениями постановления Губернатора Ямало-Ненецкого автономного округа от 22 сентября 2022 года № 125-ПГ «Об Инструкции о порядке обращения со служебной информацией ограниченного доступа».

    Вместе с базой ППО Субъект бухгалтерского учета должен предоставить регистры бухгалтерского учета, указанные выше, для осуществления Исполнителем проверки корректности загруженных данных.

    Лицензирование ПО

    Исполнителю необходимо самостоятельно приобрести лицензии на сертифицированные программные средства построения VPN и межсетевого экранирования ПО «ViPNet Client 4.х (КС2)» (производитель ОАО «ИнфоТеКС», Российская Федерация).

    Период переноса данных¶

    Исполнитель должен осуществить перенос в Систему данных из ППО на дату предоставления дистанционного доступа сотрудникам Исполнителя к базе данных ППО или передачи базы данных Исполнителю.

    Важно

    Данные за период с даты предоставления дистанционного доступа сотрудникам Исполнителя к базе данных ППО или предоставления Исполнителю базы данных исходного ППО до даты окончания переноса в Систему и выверки данных должны вноситься сотрудниками Субъекта бухгалтерского учета самостоятельно.

    Технические ограничения¶

    Оказание услуг по переносу данных должно быть осуществлено без замены или модернизации текущей технической платформы и принципов работы с Системой.

    Требования к порядку выполнения переноса данных¶

    Порядок выполнения переноса в Систему данных из ППО, используемого до момента переноса данных в Систему с соблюдением конфиденциальности переносимых сведений, должен соответствовать следующим требованиям:

    1. Перенос данных справочников и классификаторов¶

    Перенос данных справочников и классификаторов должен осуществляться в части позиций, задействованных при переносе остатков по счетам бухгалтерского (бюджетного) учета и хозяйственных операций на дату предоставления базы данных ППО.

    2. Перенос остатков по счетам бухгалтерского (бюджетного) учета¶

    Перенос остатков по счетам бухгалтерского (бюджетного) учета должен осуществляться на 01 января 2025 года в разрезе аналитических признаков, предусмотренных требованиями Приказа 157н и учетной политикой Субъекта бухгалтерского учета, кодов видов финансового обеспечения (18 разряд 26-тизначного бухгалтерского счета), кодов бюджетной классификации.

    Важно

    В случае, если в ППО отсутствуют данные в разрезе какого-либо аналитического признака, в Системе он не заполняется при переносе данных.

    Перенос оформляется первичным учетным документом Бухгалтерская справка (ф.0504833).

    3. Перенос хозяйственных операций за 2025 год¶

    Перенос хозяйственных операций за 2025 год (с 01 января 2024 года до даты предоставления базы данных ППО Исполнителю) должен осуществляться в разрезе аналитических признаков, предусмотренных требованиями Приказа 157н и учетной политикой Субъекта бухгалтерского учета, кодов видов финансового обеспечения (18 разряд 26-ти значного бухгалтерского счета), кодов бюджетной классификации.

    Важно

    В случае, если в ППО отсутствуют данные в разрезе какого-либо аналитического признака, в Системе он не заполняется при переносе данных.

    В Систему должны быть загружены все первичные документы, регистры бухгалтерского учета, сформированные за период с начала года до даты предоставления базы данных ППО Исполнителю.

    Перенос оформляется первичным учетным документом Бухгалтерская справка (ф.0504833).

    4. Перенос справочника нефинансовых активов¶

    Перенос справочника нефинансовых активов должен осуществляться по следующим реквизитам:

    • наименование объекта;
    • материально-ответственное лицо;
    • код бюджетной классификации;
    • ОКОФ;
    • амортизационная группа;
    • срок полезного использования;
    • дата ввода в эксплуатацию;
    • местонахождение;
    • единица измерения;
    • инвентарный номер;
    • завод-изготовитель;
    • заводской, реестровый, иной номер;
    • индивидуальные характеристики (сведения) объектов;
    • балансовая стоимость;
    • подразделения УФНС России по Ямало-Ненецкому автономному округу;
    • ОКПО;
    • ОКТМО.

    Требование к истории

    Сведения о нефинансовых активах, числящихся в учете на дату передачи базы Исполнителю, должны содержать полную историю зарегистрированных по ним операций, с момента принятия нефинансового актива к учету.

    5. Перенос данных по кадровому учету и расчетам по оплате труда¶

    Перенос данных по кадровому учету и расчетам по оплате труда должен осуществляться по следующим объектам:

    Данные по сотрудникам¶
    • персональные данные: ФИО, дата рождения, Пол, Документ, удостоверяющий личность, СНИЛС, ИНН, Гражданство, Наличие инвалидности;
    • табельный номер;
    • дата приема на работу, дата увольнения;
    • номер карточки/счета для зачисления заработной платы;
    • вычеты на детей, собственные вычеты;
    • составные части месячного денежного содержания, заработной платы на дату передачи базы данных Исполнителю;
    • приказы об изменении месячного денежного содержания или заработной платы с указанием документа-основания;
    • адрес по прописке.
    Прочие данные¶
    • штатная структура Субъекта бухгалтерского учета на момент даты представления данных Исполнителю согласно базе данных ППО;
    • дополнительных параметров по должностям сотрудников;
    • периодов отпусков, больничных, иных отклонений от стандартного режима работы за 2023 - 2024 года;
    • сумм начислений по сотрудникам за 2023 - 2024 год;
    • актуальных данных об остатках, не выданных (излишне выплаченных) сумм заработной платы на 01 (первое) число месяца предоставления базы данных Исполнителем.
    6. Перенос бланков строгой отчетности¶

    Перенос бланков строгой отчетности за период с начала отчетного года по дату передачи базы Исполнителю выполняется согласно Книге учета бланков строгой отчетности (при наличии учета бланков строгой отчетности).

    Акт сверки остатков и оборотов¶

    По результатам переноса и выверки баз данных Исполнитель составляет Акт сверки остатков и оборотов, который подписывается Исполнителем и Субъектом бухгалтерского учета.

    Форма акта

    Акт сверки остатков и оборотов подписывается в трех экземплярах:

    1. для Исполнителя
    2. для Субъекта бухгалтерского учета
    3. для Заказчика

    Форма Акта сверки остатков и оборотов приведена в приложении № 3 к настоящему ТЗ.

    Порядок выполнения работ¶

    В рамках оказания услуг Исполнитель должен:

    1. Развернуть тестовую базу и экземпляр Системы на своих мощностях;
    2. Осуществить перенос данных из ППО Субъекта бухгалтерского учета;
    3. Провести выверку корректности перенесенных данных на тестовой базе с привлечением пользователей Субъекта бухгалтерского учета;
    4. Осуществить перенос данных с тестовой базы на рабочую базу в Системе;
    5. Провести выверку корректности перенесенных данных на рабочей базе с привлечением пользователей Субъекта бухгалтерского учета.
